test_payment_schema.py: [Feat7-2] This test file verifies that the payment schema correctly handles valid and invalid data. It checks that a PaymentSchema object can be properly created with valid inputs, and that missing required fields or invalid values (such as negative amounts) raise appropriate validation errors. It also ensures that partial updates using UpdatePaymentSchema work as expected without requiring all fields. Additionally, it tests the CostBreakdown model to confirm that cost values are valid and that the total is correctly calculated as the sum of subtotal, delivery fee, service fee, and tax, while invalid totals are rejected.
